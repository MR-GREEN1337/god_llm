    Contributing Guide¶

    Thank you for considering contributing to the God Module! This guide will help you get started with development contributions.

    Development Setup¶

    1. Clone the Repository¶

    git clone https://github.com/mr-green1337/god_llm.git
cd god_llm

    2. Set Up Development Environment¶

    # Create virtual environment
python -m venv venv
source venv/bin/activate  # Unix
# or
.\venv\Scripts\activate  # Windows

# Install development dependencies
pip install -e ".[dev]"

    3. Install Pre-commit Hooks¶

    pre-commit install

    Code Standards¶

    Style Guide¶

    • Follow PEP 8 guidelines
    • Use type hints for all functions
    • Maximum line length: 88 characters (Black formatter)
    • Docstring format: Google style
    def example_function(param1: str, param2: int) -> bool:
    """Does something useful with the parameters.

    Args:
        param1: Description of param1
        param2: Description of param2

    Returns:
        Description of return value

    Raises:
        ValueError: Description of when this error occurs
    """
    pass

    Testing¶

    • Write unit tests for all new features
    • Maintain minimum 90% test coverage
    • Place tests in tests/ directory
    # tests/test_example.py
import pytest
from god_llm.core import God

def test_expansion():
    god = God(llm=MockLLM())
    result = god.expand("test prompt")
    assert isinstance(result, str)
    assert len(result) > 0

    Pull Request Process¶

    1. Fork and Branch 

      git checkout -b feature/your-feature-name
# or
git checkout -b fix/your-fix-name

    2. Commit Messages 

      # Format:
# type(scope): description

git commit -m "feat(core): add new expansion algorithm"
git commit -m "fix(templates): correct prompt formatting"

    3. Testing 

      # Run tests
pytest

# Check coverage
pytest --cov=god_llm tests/

    4. Documentation

    5. Update relevant documentation
    6. Add docstrings for new functions
    7. Include example usage

    """Custom error for specific cases."""
    pass

def example_function():
    try:
        # Implementation
        pass
    except Exception as e:
        raise CustomError(f"Meaningful error message: {str(e)}")

    Release Process¶

    1. Version Bump 

      # Update version in setup.py
# Update CHANGELOG.md
git commit -m "chore: bump version to X.Y.Z"

    2. Create Release 

      git tag vX.Y.Z
git push origin vX.Y.Z

    3. Build and Upload 

      python setup.py sdist bdist_wheel
twine upload dist/*
